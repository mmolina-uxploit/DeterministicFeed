# Documentación Conceptual del Proyecto

Esta carpeta contiene la documentación conceptual y arquitectónica del proyecto `DeterministicFeed`.

El propósito de estos documentos no es describir el código línea por línea, sino **explicar las ideas, decisiones y principios que gobiernan su diseño**.


---

## Contenido

Los documentos clave que estructuran el proyecto son:

- [**ARCHITECTURE.md**](ARCHITECTURE.md)  

Describe las decisiones arquitectónicas del sistema, detallando su implementación del patrón **Clean Architecture**. Explica los principios de diseño de estado y los trade-offs asumidos, respondiendo al **por qué** el sistema está construido de esta manera.

- [**TDD-AND-STATE-DRIVEN-DESIGN.md**](TDD-AND-STATE-DRIVEN-DESIGN.md)  

Explica el rol del Desarrollo Guiado por Pruebas (TDD) no solo como una herramienta de validación, sino como un **mecanismo de diseño** en una arquitectura guiada por estado. Muestra cómo los tests influyeron directamente en la forma del dominio y sus APIs.
